An **LLM-maintained wiki** about **agentic engineering** — building and operating LLM agents (Spring AI, MCP, RAG, agent security, agent workflows).

It follows [Karpathy's "LLM Wiki" pattern](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f): an LLM incrementally builds and maintains a persistent, interlinked set of notes — a *compounding* artifact — instead of re-deriving knowledge from raw documents on every query. Humans curate sources and ask questions; the LLM does the summarizing, cross-referencing, and bookkeeping.
