Day 1 of a 2-day AI-Assisted / Agentic Coding workshop for mobile/Android engineers, run remotely. Morning: AI war stories, the model & harness hierarchy, "how the machine actually works" (statelessness, hallucination, the dumb zone), privacy/data-governance. Afternoon: a hands-on deep-dive into `AGENTS.md` discipline, skills, feedback loops, and grilling a vague ticket. Driven through GitHub Copilot CLI as the contracted harness, with Claude Code shown for contrast.

The recurring spine: **stop being the servant of the AI** — don't copy-paste; give the agent a feedback loop and the tools to check its own work — and **protect your context window** (the dumb zone is where slop, forgetting, and dropped prod databases happen).

Key concepts introduced or reinforced: [[model-hierarchy]] (Opus/Sonnet/Haiku/GPT as experienced hires), [[harness]] (CLI > IDE plugin), [[chop-ai-waiter]] anti-pattern, [[cognitive-debt]] / vibe-coding collapse, [[archunit-drift-control]], [[multi-model-review]] aggregation pipeline, [[skills-governance]] (megalomania vs dichotomy), [[memory-layers]], [[agents-md-discipline]] (anti-dumping, the seed-of-heresy Socratic loop, sub-folder injection, retrospect-every-sprint), [[custom-agent]], [[data-governance]] (ZDR), [[knowledge-cutoff]], [[context7]], [[headroom]] / [[rtk]], [[prompt-caching]], [[dumb-zone]], [[sycophancy]], [[feedback-loop]], [[agentic-corporation]] (blackboard, dark factory), [[figma-mcp]], [[adb-automation]], [[cli-vs-mcp-tradeoff]] (gh CLI over GitHub MCP), [[grill-me]] / vague-ticket grilling, [[ralph-loop]], [[acceptance-test-bdd]] (business-reviewed `.feature` files), [[wispr-flow]], [[agent-skill]] front-matter as trigger.
